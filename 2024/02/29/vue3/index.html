前置知识

JavaScript-导入导出

  • 有一个有很多功能函数的js文件，我们只需要用到其中部分的函数，若导入整个文件，会让代码体积不必要的增大，增加了网页的加载时间。
  • js提供的导入导出机制，可以实现按需导入。即只导入该文件的部分函数

在js文件中导出需要用到的函数

//简单的展示信息
function simpleMessage(msg){
    console.log(msg)
}
//复杂的展示信息
function complexMessage(msg){
    console.log(new Date()+": "+msg)
}
// 批量导出
export {simpleMessage,complexMessage}

在html文件中导入

<div id="app">
    <button id="btn">点我展示信息</button>
</div>
<script type="module">
    import{complexMessage as cm} from './showMessage.js';
    document.getElementById("btn").onclick=function(){
        cm('我被点击了...');  // as重命名了
    }
</script>

注意

  • 导入导出的时候可以用as重命名

  • 导入和导出的函数名要统一（默认导出不用）

  • 默认导出：在js文件的批量导出中加上关键字Default

总结

  • 导出：

    • export 内容
    • export {内容1，内容2，…}
    • export default {…}

  • 导入

    <script type="module"> 
    import{...} from '...'
</script>

常用指令

指令 作用
v-for 列表渲染，遍历容器的元素或者对象的属性
v-bind 为html标签绑定属性值，如设置href、css等
v-if / v-else-if / v-else 条件性的渲染某元素，判定为true时渲染
v-show 根据条件展示某元素，区别在于display属性值
v-model 在表单元素上创建双向数据绑定
v-on 为html标签绑定事件

v-for

  • 语法： v-for = “(item, index) in items”

  • 参数说明：

    • items： 遍历的数组

    • item： 遍历出来的元素（自己语义化取名）

    • index： 下标，从0开始；可以省略，省略的index语法：

      v-for = “item in items”

注意：遍历的数组必须在data中定义；要想让哪个标签循环展示多次，就在哪个标签上使用v-for指令。

v-bind

  • 作用： 动态为html标签绑定属性值，如设置href，src，style样式等
  • 语法： v-bind：属性名=“属性值”
  • 简化： : 属性名=“属性值”
  • 注意： v-bind绑定的数据，必须在data中定义

v-if 和 v-show

  • 作用： 这两类指令，都是用来控制元素的显示与隐藏的
  • v-if
    • 语法： v-if=“表达式”，表达式为true显示，false隐藏
    • 其他：可以配合v-else-if / v-else进行链式调用条件判断
    • 原理：基于条件判断，来控制创建或移除元素节点（条件渲染）
    • 适用场景：要么显示，要么不显示，不频繁切换的场景
  • v-show
    • 语法： v-show=“表达式”
    • 原理： 基于css样式display属性来控制显示与隐藏
    • 适用场景：频繁切换显示或隐藏的场景
  • 重点：若条件不成立，v-if不会创建该元素节点，v-show则是在创建该元素结点之后进行判断，条件不成立设置display为false不显示。

v-on

  • 作用： 为html元素绑定事件（如click）
  • 语法： v-on：事件名=“函数名” 或 @事件名=“函数名”
  • 注意： 函数需要定义在methods选项内部

v-model

  • 作用：在表单元素上使用，双向数据绑定（数据和视图其一发生变化，对方也会跟着变化）。可以方便地获取或设置表单项数据
  • 语法： v-model = “变量名”
  • 注意： v-model中绑定的变量，必须在data中定义

Vue生命周期

  • 生命周期： 指一个对象从创建到销毁的整个过程
  • 生命周期的八个阶段：每个阶段会自动执行一个生命周期方法（钩子函数），让开发者有机会在特定的阶段执行自己的代码。
状态 创建周期
beforeCreate vue实例创建前
created vue实例创建后
beforeMount vue实例载入前
mounted vue实例挂载完成
beforeUpdate vue实例数据更新前
updated vue实例数据更新后
beforeUnmount vue实例组件销毁前
unmounted vue实例组件销毁后
  • 主要记住mounted，我们一般会在mounted函数中发送请求拿到页面需要展示的数据，以便将来可以把这些数据渲染到页面上进行展示
  • vue生命周期典型的应用场景：在页面加载完毕时，发起异步请求，加载数据，渲染页面
  • 钩子函数和data，methods是并列同级的，需要单独声明。

import axios from 'axios';
// 定义一个变量, 记录公共的前缀, baseURL
const baseURL = 'http://localhost:8080';
const instance = axios.create({baseURL});
// 获取所有文章数据
export async function articleGetAllService(){
    // 发送异步请求，获取所有文章数据
    // 同步等待服务器相应的结果,并返回, async await
    return await instance.get('/article/getAll')

}
// 根据文章分类和发布条件进行搜索
export async function articleSearchService(conditions) {
    return await instance.get('/article/search', { params: conditions})
        .then((result) => {
            return result.data;
        }).catch((err) => {
            console.log(err);
        });
}

拦截器

  • 在请求或响应被then或catch处理前拦截它们

// 定义一个变量, 记录公共的前缀, baseURL
const baseURL = 'http://localhost:8080';
const instance = axios.create({ baseURL });

// 添加响应拦截器
instance.interceptors.response.use(
    result=>{
        return result.data;
    },
    err=>{
        alert('服务异常');
        // 异步的状态转化成失败的状态
        return PromiseRejectionEvent.reject(err);
    }
)
// 将该实例对外暴露
export default instance;

前端开发步骤

跨域问题

  • 由于浏览器的同源策略限制，向不同源（不同协议、不同域名、不同端口）发送ajax请求会失败

  • 比如浏览器所在的源为 http://localhost:5173 , 发送ajax的源是http://localhost:8080, 由于是不同端口，会发送失败

  • 只有浏览器会存在跨域问题

解决方法

  • 代理：在服务器端设置一个代理服务器，所有的请求都先发送到代理服务器，然后由代理服务器转发给目标服务器。这样，浏览器与代理服务器之间的请求就不会受到同源策略的限制。（服务端之间不存在跨域问题）

  • 举例（后端代理）：前端（源为5173）发送一个请求到后端服务器（5173的服务端）。然后，后端服务器作为客户端，向另一个服务（源为8080）发送请求。这样，前端不需要直接访问8080的服务，而是通过5173的服务端作为中介来间接访问。

  • 示例：

    • request.js 中配置baseURL的值为 /api

    • vite.config.js中配置

    server:{
    proxy: {
      '/api':{
          target: 'http://localhost:8080', // 后台服务器所在源
          changeOrigin: true,  //true ,修改源
          rewrite: (path) => path.replace(/^\/api/, '') // api替换为''
      }
    }
  }

===和!==

  • “===” 运算符用于比较两个值是否严格相等。这意味着它不仅比较两个值的内容，还比较它们的类型。如果两个值的内容和类型都相同，则它们被认为是严格相等的。
  • “==” 运算符是宽松相等运算符，它在比较时会执行类型转换，有时可能会导致意料之外的结果。
  • !==同理

函数表达式和普通函数

  • 在大多数情况下，函数声明和函数表达式可以互换使用，选择哪种方式主要取决于你的个人偏好和代码风格。箭头函数表达式提供了更简洁的语法和不同的 this 上下文绑定规则，因此在现代 JavaScript 编程中越来越受欢迎。

  • 区别

    1. this 关键字的行为：
      • 在箭头函数中，this 关键字不绑定自己的 this，而是继承自包围它的函数或全局作用域。
      • 在普通函数中，this 的值取决于函数的调用方式。
    2. 简洁性：
      • 箭头函数通常更简洁，不需要使用 function 关键字和 {} 来包围函数体。
    3. 错误处理：
      • 在箭头函数中，你不能使用 new 关键字来构造一个对象，因为它没有自己的 this 和 prototype。
      • 在普通函数中，这是没有限制的。

  • 箭头函数（函数表达式的一种）

const clearRegisterData = ()=>{
    registerData.value = {
        username: '',
        password: '',
        rePassword: '',
    }
}

            rePassword: '',
    }
}

富文本编辑器

文章内容需要使用到富文本编辑器，这里咱们使用一个开源的富文本编辑器 Quill

官网地址： https://vueup.github.io/vue-quill/

安装：

npm install @vueup/vue-quill@latest --save

导入组件和样式：

import { QuillEditor } from '@vueup/vue-quill'
import '@vueup/vue-quill/dist/vue-quill.snow.css'

页面长使用quill组件：

<quill-editor
              theme="snow"
              v-model:content="articleModel.content"
              contentType="html"
              >
</quill-editor>

样式美化：

.editor {
  width: 100%;
  :deep(.ql-editor) {
    min-height: 200px;
  }
}
